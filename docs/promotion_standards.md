## 引擎参与机制

### 调度入口
所有引擎由 `cognitive_loop` 统一调度：
- priority 越高越优先调度
- experimental 引擎在 EXPLORE 模式下会被调度
- 间隔到了且引擎处于 running 状态才会执行

### 事件驱动参与
以下事件可触发引擎参与：
- `openclaw.session_started` → openclaw_memory
- `task.completed` → skill_evolution
- `task.failed` → skill_evolution
- `skill.executed` → skill_evolution
- `engine.executed` → skill_evolution

### 手动触发
```bash
# 手动运行某个引擎
cd /home/xiaoxin/fuxi && python3 -c "
from fuxi.engines.base import get_engine_registry
eng = get_engine_registry()._engines['openclaw_memory']
eng._execute()
print(eng.run())
"
```

## 转正流程

1. **观察期**：引擎运行 ≥50次，期间统计各项指标
2. **评估期**：检查是否达到所有阈值
3. **决策期**：
   - 全部达标 → 转正（移除 experimental 标记）
   - 未达标 → 分析原因，优化后继续观察
4. **归档期**：连续30天未达标且无法修复 → 标记为 deprecated

## 评分计算

```
总分 = (连续运行分 × 0.3) + (功能产出分 × 0.4) + (稳定性分 × 0.3)

连续运行分 = min(运行次数/目标次数, 1.0) × 100
功能产出分 = min(实际产出/目标产出, 1.0) × 100
稳定性分 = (1 - 错误率) × 100

转正线：总分 ≥ 75
```
